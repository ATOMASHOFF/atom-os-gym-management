# ATOM FITNESS — Operations & Backup Runbook

## 🗄️ Database Backups

### Render.com (recommended hosting)
Render Starter DB includes **daily automated backups** with 7-day retention.

To manually trigger a backup:
```bash
# Via Render dashboard: Database → Backups → Create Backup
# Or via Render CLI:
render backups create --service-id <your-db-service-id>
```

To restore:
```bash
# Render dashboard: Database → Backups → select backup → Restore
```

### Self-hosted PostgreSQL backup
```bash
# Daily backup script — add to crontab: 0 2 * * * /opt/backup-atom-fitness.sh
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups/atom-fitness"
mkdir -p $BACKUP_DIR

pg_dump $DATABASE_URL \
  --format=custom \
  --compress=9 \
  --file="$BACKUP_DIR/atom_fitness_$DATE.dump"

# Keep only last 30 days
find $BACKUP_DIR -name "*.dump" -mtime +30 -delete

# Upload to S3 (optional)
# aws s3 cp "$BACKUP_DIR/atom_fitness_$DATE.dump" s3://your-bucket/backups/
echo "Backup complete: atom_fitness_$DATE.dump"
```

### Restore from backup
```bash
pg_restore --clean --if-exists -d $DATABASE_URL backup_file.dump
```

## 🔧 Common Operations

### Add a new gym
```sql
-- Direct DB (use with care)
INSERT INTO gyms (name, slug, owner_name, owner_email, owner_phone, address)
VALUES ('New Gym', 'new-gym', 'Owner Name', 'owner@gym.com', '+91-...', 'Address');
```

Then create the admin user via `/api/public/register` + approve them, or directly:
```sql
INSERT INTO members (name, email, password_hash, role, gym_id, status, qr_token)
VALUES ('Admin Name', 'admin@email.com', '<bcrypt-hash>', 'admin', <gym_id>, 'active', gen_random_uuid()::text);
```

### Expire overdue subscriptions (run weekly)
```sql
UPDATE subscriptions
SET status = 'expired', updated_at = NOW()
WHERE status = 'active' AND end_date < CURRENT_DATE;
```

### Find members whose subscription expires this week
```sql
SELECT m.name, m.email, m.phone, s.end_date, p.name as plan, g.name as gym
FROM subscriptions s
JOIN members m ON s.member_id = m.id
JOIN membership_plans p ON s.plan_id = p.id
JOIN gyms g ON s.gym_id = g.id
WHERE s.status = 'active'
  AND s.end_date BETWEEN CURRENT_DATE AND CURRENT_DATE + 7
ORDER BY s.end_date;
```

### View audit log for a member
```sql
SELECT action, old_data, new_data, created_at, actor_role
FROM audit_logs
WHERE entity = 'members' AND entity_id = <member_id>
ORDER BY created_at DESC;
```

## 🚨 Incident Response

### API is down
1. Check Render dashboard for service status
2. `curl https://atom-fitness-api.onrender.com/health`
3. Check logs for crash reason
4. If DB issue: check DB service is running
5. Rollback: Render dashboard → Deploys → previous deploy → Rollback

### Member can't log in
1. Check member status in DB: `SELECT status, is_active FROM members WHERE email='...'`
2. May be `pending` (needs approval), `suspended`, or `inactive`
3. Admin can update via Members page or SQL

### Performance degraded
1. Check `EXPLAIN ANALYZE` on slow query
2. Check DB pool: look for `pool exhausted` in logs
3. Check if any missing indexes: run `pg_stat_user_indexes`
